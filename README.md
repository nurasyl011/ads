📌 Project Overview
Граф — это структура данных, состоящая из вершин (Vertex) и рёбер (Edge).
В данной работе граф представлен через список смежности.
Я реализовали два алгоритма обхода графа:
Breadth-First Search (BFS) — поиск в ширину
Depth-First Search (DFS) — поиск в глубину

📌 Class Descriptions
Vertex — хранит уникальный идентификатор вершины.
Edge — хранит начало и конец ребра.
Graph — реализует список смежности, методы добавления вершин и рёбер, а также BFS и DFS.
Experiment — запускает обходы и измеряет время выполнения.
Main — создаёт графы разных размеров (10, 30, 100 вершин), запускает эксперименты и выводит результаты.

📌 Algorithm Descriptions
BFS
Использует очередь.
Обходит граф по уровням.
Применяется для поиска кратчайшего пути.
Сложность: O(V + E).

DFS
Использует рекурсию или стек.
Уходит в глубину, пока есть непосещённые вершины.
Применяется для топологической сортировки, поиска компонент связности.
Сложность: O(V + E).

Время растёт линейно с увеличением числа вершин.
BFS и DFS показывают близкие результаты, что соответствует теории (O(V+E)).
BFS иногда быстрее на малых графах, DFS — на средних.

📌 Reflection
В ходе работы я:

Научился реализовывать граф через список смежности.
Освоил алгоритмы BFS и DFS и увидел их различия на практике.
Убедился, что сложность действительно соответствует O(V+E).
Столкнулся с трудностью в организации кода и правильном измерении времени, но решил её через System.nanoTime().
Вывод: BFS лучше для поиска кратчайшего пути, DFS — для анализа глубины и структуры графа.
